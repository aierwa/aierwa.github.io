MapReduce单词计数实例

发表于 2018-05-01 | 分类于 Java

目的

快速安装配置hadoop，本地模式实践，通过单词计数了解mr的原理。

准备

  1. Linux系统 | CentOS
  2. Java 1.8
  3. ssh，管理远程hadoop进程时sshd必须正在运行
  4. hadoop-2.8.1

安装

CentOS

先安装VMware虚拟机，再安装linux，过程略。

Java

  1. 下载：curl -O https://download.java.net/java/jdk8u172/archive/b03/binaries/jdk-8u172-ea-bin-b03-linux-x64-18_jan_2018.tar.gz
  2. 解压：tar -xzvf jdk-8u172-ea-bin-b03-linux-x64-18_jan_2018.tar.gz -C /opt/modules
  3. 配置：
  • vim /etc/profile
    export JAVA_HOME=/opt/modules/java1.8.0_172
    export CLASSPATH=$JAVA_HOME/lib/
    export PATH=$PATH:$JAVA_HOME/bin
  • source /etc/profile
  • java –version

ssh

是否启动（有sshd说明启动）：sudo ps -e | grep ssh
启动：sudo service ssh start

hadoop

  1. 下载：curl -O https://mirrors.tuna.tsinghua.edu.cn/apache/hadoop/common/hadoop-2.8.1/hadoop-2.8.1.tar.gz
  2. 如果https下载不了，则先升级：yum update nss
  3. 解压：
  4. 配置：

    vim etc/hadoop/hadoop-env.sh
    修改：export JAVA_HOME=/opt/modules/jdk1.8.0_172

伪分布式hadoop实例

配置

etc/hadoop/core-site.xml

etc/hadoop/hdfs-site.xml

设置ssh免密登录

ssh localhost：看需不需要密码，如果每次都要，则执行：
ssh-keygen -t rsa -P ‘’ -f ~/.ssh/id_rsa
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
chmod 0600 ~/.ssh/authorized_keys

运行

  • 格式化文件系统：bin/hdfs namenode –format
  • 开启namenode和datanode进程：sbin/start-dfs.sh
  • 查看：jps
    如果datanode未启动，解决：将/tmp/hadoop-username/dfs/data/current/VERSION中的clusterID值改为和/tmp/hadoop-username/dfs/name/current/VERSION中的一样。
  • 生成必须的文件夹：bin/hdfs dfs –mkdir /user
    bin/hdfs dfs –mkdir /user/
  • 复制输入文件到分布式文件系统中：bin/hdfs dfs –put input input



  • 执行：bin/hadoop jar test/map-reduce-demo-1.0-SNAPSHOT.jar WordCountMapReduce input output
  • 查看结果：bin/hdfs dfs –cat output/*

  • 停掉进程：sbin/stop-dfs.sh
