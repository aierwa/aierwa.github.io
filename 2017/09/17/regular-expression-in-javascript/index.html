关于正则

正则表达式，Regular Expression，是由一个字符序列形成的搜索模式。也就是用它可以用搜索模式来描述你要查询的内容。

在JS中，就是RegExp对象。

模式及匹配

参考手册：w3school

修饰符：
即i，g，m

方括号：
查找某个范围，如[abc]，[^abc]，[A-Z]，[A-z0-9]

元字符：
拥有特殊含义的字符。如\w, \W, \s, \d, \s等。

量词：
即匹配多少个。如n+, n*, n?, n{3}, n{3,7}, n{3,}, ?=n, ?!n等等。

捕获组：
正则表达式之捕获组/非捕获组介绍

使用小括号指定一个子表达式后，匹配这个子表达式的文本(也就是此分组捕获的内容)可以在表达式或其它程序中作进一步的处理（如exec等方法常用）。

  • (pattern) ：匹配pattern并捕获结果，自动设置组号。
  • (?pattern)：匹配pattern并捕获结果，设置name为组名。
  • \num：对捕获组的反向引用。其中 num 是一个正整数。
  • \k：对命名捕获组的反向引用。其中 name 是捕获组名。

非捕获组：

即：使用了小括号，但不将之作为捕获组。

  • (?:pattern)：匹配pattern，但不捕获匹配结果。如industr(?:y|ies)。
  • (?=pattern)：零宽度正向预查，不捕获匹配结果。
    Windows (?=95|98|NT|2000)，匹配”Windows2000”中的”Windows”不匹配”Windows3.1” 中的 “Windows”。
  • 等等…

关于多行模式（m）：

主要是对有^和$符号匹配时有影响，当字符串中有换行符（\n或\r）时，如果正则表达式没有m模式，则字符串会当作一行进行处理，此时的^仅匹配字符串首位置；如果有m模式，则此时的^可以匹配字符串首位置以及\n或\r的后一位置。

RegExp 对象属性

  • global：RegExp 对象是否具有标志 g。
  • ignoreCase： RegExp 对象是否具有标志 i。
  • multiline：RegExp 对象是否具有标志 m。
  • lastIndex：一个整数，标示开始下一次匹配的字符位置。
  • source：正则表达式的源文本。

RegExp 对象方法

reg.test(str)

检索字符串中指定的值。返回 true 或 false。

reg.exec(str)

返回一个数组，其中存放匹配的结果。如果未找到匹配，则返回值为 null。

exec() 方法的功能非常强大，它是一个通用的方法。

非全局（无g）匹配情况下：匹配到了返回数组，数组项第0个元素是与正则相匹配的文本，第1个元素是与正则的第1个子表达式相匹配的文本（如果有的话），后面依次类推。除此之外，该数组还有两个属性，index属性声明匹配文本出现的位置，input属性存放的是被检索的字符串string。
非全局下，返回数组结果与String.match()返回是相同的。

全局（g）匹配情况下：行为稍复杂。它会在正则的lastIndex属性指定的字符处开始检索字符串string。当 exec() 找到了与表达式相匹配的文本时，lastIndex 属性设置为匹配文本的最后一个字符的下一个位置，这就是说，可以通过反复调用 exec() 方法来遍历字符串中的所有匹配文本，直到查找完返回null为止，并把 lastIndex 属性重置为 0。

总结：

返回一个数组，未找到则返回null

数组内容：

  • 0：与正则相匹配的文本
  • 1：第一个子表达式
  • 2：第二个子表达式
  • n：第n个子表达式
  • index：匹配到的位置
  • input：被检索的字符串s

若非全局匹配，则就返回一次这个数组；

若全局匹配，则可重复调用exec()方法，直到返回null为止！

支持正则的 String 对象相关方法

str.search(reg)

若查到，返回起始位置，否则返回 -1。没有全局查找功能。

str.match(reg)

若非全局查找，则和exec()结果一样：

若全局查找，返回所有找到的，此时子捕获就没用了

str.split(reg)

分割字符串，返回数组。

str.replace(reg, func)

功能强大的方法，可以利用子捕获组。

第一个参数reg可以是全局（全部替换），也可以非全局（替换匹配到的第一个）。第二个参数可以是字符串，或者是返回字符串的一个函数。

  1. 元字符 . 表示除换行符以外的任意字符，若要匹配任意字符，可以使用[\s\S]。
