关于CSS

历史、版本

Web的早期，几乎是HTML的一己之力，内容，显示，交互等都一手操办。但当web喷涌的时候，HTML力不从心了，从此诞生了CSS和JS。
以前控制样式都是通过HTML标签来实现的，如<font>控制字体。页面内容增加后，开始冗杂，非结构化，难以维护。
于是先是两三个人商量着如何解决这个问题，然后成立一个工作组（W3C组织里面），开始研究、制定草案。

从1994年提出层叠样式表（Cascading Style Sheets，CSS），到1996年年底，工作组完成了第一份正式标准，即CSS1。目前来说CSS1已经被认为是过时的了。
（注意：CSS没有传统意义的版本说法，而是“级别/水平”（levels），更高级别的CSS是低级别CSS的超集。CSS1即为CSS Level 1，CSS3即为CSS Level 3。）

1998年CSS2就出来了，在CSS2的实施实践过程中，工作组发现规范中的诸多问题，于是选择定义CSS Level 2 Revision 1（CSS2.1）。也就是说，CSS2.1是CSS2的修订版，如果两个版本之间有任何冲突，CSS2.1会有明确的定义。现在使用最广泛的就是CSS2.1。

CSS3以CSS2.1规范作为核心，在CSS2模块基础上按模块建立。也就是说，CSS3是按各属性模块分别定义的，比如颜色模块，边框模块，动画模块等，“CSS3”这个术语仅用来区别之前的单一模块版本。如果CSS2中没有相关模块就从level 1开始，比如CSS Transforms Module Level 1，Selectors API Level 1；如果CSS2中有的模块就从level 3开始，乃至4或更高，比如Selectors Level 3，Media Queries Level 4。当然，这些模块有些已经完成并在推荐阶段，有些在草案阶段。由于是模块单独建立，所以Selectors Level 4是有可能在CSS Line Module Level 3之前完成的。随着每个模块的完成，它们将被插到现有的CSS2.1系统中。

CSS4，如前所述，CSS3已经是按模块来定义了，所以哪里有什么CSS Level 4啊！

选择器

选择器是CSS的核心，要对文档应用样式，首先就得定位元素。

哪些选择器？

通配符： *
适配所有元素

ID选择器： #span-1

类选择器： .center
.urgent.warning 指同时包含这两个值的元素。（IE7-不支持）

元素选择器： div div,p

后代选择器： div p

子选择器： div > p
注意：查找顺序是从右至左！

属性选择器：

  • 含某属性： a[href] p[class] *[title] a[href][title]
  • 具体属性： a[href="aierwa.github.io"] p[class="urgent"]
    即class属性值等于urgent的p元素，“urgent red”这种不行！
  • 部分属性值： p[class~="urgent"]
    具有class值“urgent”的p，“urgent red”这种也能匹配，等价于.urgent。
  • 开始子串属性值： p[class^="test-"] (css3)
  • 结束子串属性值： a[href$=".pdf"] (css3)
  • 任意子串属性值： a[href*="port"] (css3)

相邻兄弟选择器： h1 + p

伪类/伪元素选择器： :link :visited :focus :hover :active :first-child :first-letter :first-line :lang(fr) :before :after
链接a的顺序：LVHA

字体、文本

字体

字体系列
如Times,Verdana,Helvetica,Arial这些都是一种特定的字体系列，其下包含好几种变形字体。CSS单独定义了几种通用字体系列：

  • serif： 字体成比例，有上下短线。如Times, Georgia, New Century Schoolbook.
  • sans-serif： 字体成比例，无上下短线。如Helvetica, Geneva, Verdana, Arial, Univers.
  • monospace： 字体不成比例，模拟打字机字体，字母宽度相同，如Courier, Courier New, Andale Mono.
  • cursive： 模范人的手写体，字母主要会有一些曲线。如Zapf Chancery, Author, Comic Sans.
  • fantasy： 未知的特征。

文本

可完全代替之前的一些HTML标记，如<font>，<center>。

  • text-indent
    首行缩进，应用于块级元素。如：p {text-indent: 3em;}。也可为负值，实现“悬挂缩进”。

  • text-align
    水平对齐，应用于块级元素。left,center,right,justify。

  • line-height
    文本行基线间的距离。line-height与字体大小之差就是行间距（文本上下各应用差的一半），应用于所有元素。具有继承性，最好设置缩放因子，继承的是缩放因子值，而非计算值，一般的行高值为字体大小的1.2倍。
    这里先来说说文本行、行内框和行框：文本行就是一行文本；行内框指文本行中的每个元素（单个字、单词）都会产生的框，其内容区由字体大小确定，如果没有其他因素，行内框就完全等于内容区，但是line-height产生的行间距可以使行内框高度增加；行框是能包含整个一行文本的框，即足以包含最高行内框的顶端和最低行内框的底端。

  • vertical-align
    垂直对齐，应用于行内元素和表单元格，不影响块级元素。有值：baseline,sub,super,top,text-top,middle,bottom,text-bottom,以及长度值。

  • word-spacing,letter-spacing
    单词间距、字母间距

  • text-transform
    文本转换，uppercase,lowercase,capitalize，最后这个为首字母大写。

  • white-space
    此属性影响源文档中空格、换行和tab字符的处理，应用于所以元素。
    默认情况下，也即normal的情况，文本中连续的空白符会合并为一个空格（所以需要多个则写&nbsp;）；
    pre：保留空格及换行符，好比<pre>标签，不允许自动换行；
    nowrap：禁止换行，除非使用<br>标签，不允许自动换行；
    pre-line：合并空白符，保留换行符，允许自动换行；
    pre-wrap：保留空白符，保留换行符，允许自动换行。

基本视觉（块级、行内）

这是关于CSS如何表现的理论知识，对于了解CSS的行为有帮助。

一些概念

  • 正常流
    即我们传统的文档布局：从左至右，上下至下。要让一个元素不在正常流中，唯一办法就是浮动或定位。（本章只讨论正常流中的元素）

  • 非替换元素
    元素的内容即在文档中。如p、span元素

  • 替换元素
    元素作为其他内容的占位符，如img、input元素

  • 块级元素
    指段落、标题或div之类，正常流中，其框之前和之后生成“换行”。指定display:block可使元素生成块级框。

  • 行内元素
    指span、strong之类，之前之后不生成“行分隔符”。

  • 包含块
    包含块就是一个元素的“布局上下文”，由++最近的块级祖先元素、表单元格或行内块祖先框的内容边界++构成，注意，是内容边界。

  • 匿名文本
    即未包含在行内元素的文本。

  • em框
    是在字体中定义的，也称为字符框。font-size属性就是设置字体em框大小的（注意font-size不是只某个字母的大小哦！）。

  • 行内框
    内容区加行间距形成的框（针对元素来说），其值就为line-height值。

  • 行框
    包含该行中所有行内框的最高点和最低点形成的最小框（针对一行来说）。

块级元素

水平方向：有七个属性：margin-left,border-left,padding-left,width,padding-right,border-right,margin-right. CSS2.1下width指的是内容区的宽度，而可见区域（即设置背景的区域）宽度是包含内容width、padding和border部分的。在CSS3中可通过box-sizing属性来改变width的指定。
并且有一个原则：这7个属性的值加在一起必须是元素包含块的宽度。，这往往就是该元素的父块级元素的width值。

width/height和margin可以设置为auto值，padding不能！
对于width,margin-left,margin-right三个值，若仅一个为auto，则自动计算其长度；若两个margin为auto，则设置值相等，若width和其中一个margin为auto，则那个为auto的那个margin值自动计算为0；若三个值发生冲突，则自动视margin-right为auto。
对于margin-top,margin-bottom设置为auto，则会自动计算为0（所以通过设置margin，可水平居中一个块级元素，而不能垂直居中它）。注意：对于定位元素来说，上下margin设为auto时处理有所不同。

外边局margin可以设置为负数，但width,border,padding都不能为负！

高度height为auto时（也就是默认情况下），有个奇怪的现象：其高度会设置为其包含的子块级元素的最上边框到最下边框之间的距离，这时，子元素的外边距会“超出”包含这些子元素的元素。）；但当这个块级元素设置有内边距或边框时，其高度就会设置为其包含的子块级元素的最上外边距边界到最下外边距边界之间的距离。

垂直margin会合并为较大者。

行内元素

font-size,line-height,vertical-align会影响行内框的显示，继而影响行框，改变行间距。

padding可以拉开边框，但这并不能改变行高，并不能影响行框的生成，虽然此时若应用背景好似变高了一样。border也不影响。

可以为行内元素应用左右margin，而margin-top,margin-bottom无效。（行内元素被浮动后就可以指定上下margin了。）

盒子模型

CSS盒子模型老生常谈了，面试的时候常问，可见其重要性。它让我们可以更方便、强大地去控制布局和显示，是网页制作的基础。

由外至内：margin,border,padding,content。

首先需要说明的就是width,height属性，其值默认指content部分，不包含padding，CSS3中可指定box-sizing来控制它，比如box-sizing:border-box指明width包含padding。
另外，宽高设置对行内元素无效。

对于margin：可简写如margin: 1em,margin: 1em 2em,margin: 1em 2em 3em 4em，上右下左顺时针。正常流中垂直相邻外边距会合并为较大值（浮动元素不合并）。

border属性是border-style,border-width,border-color三个属性的简写。border-color有一个transparent值，较有用。

对于padding：若是百分数值，则是相对于父元素的width进行设置（margin也如此）。

浮动

浮动，不完全是定位，也不属于正常流。更像自成一派。
