一个小的Git项目实践

发表于 2017-08-27 | 分类于 前端

目的：走一遍Git的工作流程

1、新建文件夹并初始化Git项目


使用git init命令初始化Git项目，默认会创建主分支master。

2、新建readme并添加到暂存区index



使用vim readme新建文件，并输入name: xuxiang，然后使用git add命令将文件添加到暂存区index。

3、将改动提交到仓库


使用git commit命令将暂存区index的改动提交到版本库，-m选项可添加注释，永远建议加上。Mode 100644表示文件类型为100（regular file），文件权限644。此时简单的添加文件并提交的流程就完了。

4、改动文件并查看修改


此时readme添加age行，使用git status命令查看文件状态，它告诉我们改变还没有被准备提交，也就是说改动还没添加到暂存区中，并告诉可以用”git add …”更新即将被提交的内容，或者”git checkout – …”来放弃工作区中的改动。

使用git diff命令查看修改，显示格式是Unix通用的diff合并格式。
--- a/readme表示变动前的版本，+++ b/readme表示变动后的版本。
@@ -1 +1,2 @@，-1表示修改前是从第1行显示，+1,2表示修改后文件是从第一行开始的连续2行。

再次使用git add和git commit命令进行版本提交。

5、查看版本提交历史

此时我们有两个版本了：1-“add a readme file”；2-“add age”

使用git log查看提交历史（--oneline简洁显示），从最近的一次开始显示。
commit后面的一串十六进制数字是commit id（版本号）。
