Express入门

发表于 2017-08-28 | 分类于 前端

关于Express

Express 是一个基于 Node.js 平台的极简、灵活的 web 应用开发框架。具有丰富的 HTTP快捷方法和任意排列组合的 Connect 中间件，方便的路由管理。

也就是说，我们可以用express快速的开发出一个node应用。

Express 路由

路由，指如何定义应用的端点，以及如何响应客户端请求。

路由结构：app.METHOD(path, [callback…], callback);

其中，app是express实例（或者Router对象），METHOD是HTTP方法，path是路径，后面的是句柄。

路由路径

可以是字符串、字符串模式、正则。

  • 字符串：'/about'

  • 字符串模式：

    • '/ab?cd'：匹配acd, abcd

    • ‘/ab*cd’：匹配abxxcd, abcd

  • 正则：

    • /a/：匹配任何带有a的路径

    • /.\*fly\$/：匹配butterfly等

路由句柄

即是回调函数，可提供多个，行为类似于中间件，可以调用next()方法略过其他路由函数。

一个router对象就像一个小的应用一样，能够执行自己的中间件和路由函数。

它表现得就像中间件本身，所以你可以在app.use()，或者另一个router.use()中作为参数传入。即：app.use('/birds', birds)

router也可以使用app的get(), post(), router(), use()等方法。

响应对象

res的方法向客户端返回响应，终结请求响应的循环，如果不调用，则客户端的请求就会一直挂起。

方法：

  • res.download()：提示下载文件
  • res.end()：终结响应处理流程
  • res.json()：发送一个json格式的响应
  • res.jsonp()：发送一个支持jsonp的json格式响应
  • res.redirect()：重定向请求
  • res.render()：渲染视图模板
  • res.send()：发送各种类型的响应
  • res.sendFile()：以八位字节流的形式发送文件
  • res.sendStatus()：设置响应状态码，并将其以字符串形式作为响应体的一部分发送。
  • res.cookie()：设置cookie，res.cookie(‘name’, ‘xx’, {maxAge: 900000, httpOnly: true});
  • res.clearCookie()：清除cookie

属性：

  • res.locals：一个包含响应对象局部变量的对象，可在模板中使用。

Express 中间件

从本质上来说，一个 Express 应用就是在调用各种中间件。

简单说，中间件（middleware）就是处理HTTP请求的函数，参数为req, res, next。。它最大的特点就是，一个中间件处理完，再传递给下一个中间件。

中间件功能：

  • 执行任何代码。（在函数里面）
  • 修改请求和响应对象。（操作 req, res ）
  • 终结请求-响应循环。（比如 res.end() ）
  • 调用堆栈中的下一个中间件。（ next() ）

切记：如果当前中间件没有终结请求-响应循环，则必须调用 next()方法将控制权交给下一个中间件，否则请求就会挂起。

中间件种类：

  • 应用级中间件
  • 路由级中间件
  • 错误处理中间件
  • 内置中间件
  • 第三方中间件

应用级中间件

绑定到app对象，使用 app.use() 和 app.METHOD()。

路由级中间件

用法与app类似，用来处理路由请求。

错误处理中间件

注意：错误处理中间件有4个参数，即使不需要，都要写在参数中，即err, req, res, next，否则会被认为是常规中间件。

可以定义多个错误处理中间件，next(err)会直接跳过中间的句柄，除了那些用来处理错误的句柄。

内置中间件

express.static(root, [options])是目前唯一的内置中间件。负责在应用中托管静态资源。

第三方中间件

如cookie-parser等等

关于next

我们在定义express中间件函数的时候都会将第三个参数定义为next，next函数主要负责将控制权交给下一个中间件，如果当前中间件没有终结请求，并且next没有被调用，那么请求将被挂起，后边定义的中间件将得不到被执行的机会。

如果我们定义的中间件终结了本次请求，那就不应该再调用next函数，否则就可能会出问题（比如两个中间件都执行了res.send()就会报错。）

req 对象

req 即是 HTTP request 对象，可以获取如查询字符串、参数、请求体、HTTP头等等。

属性

  • req.app：正在使用的express实例
  • req.baseUrl：挂载在路由实例上的URL路径（比如前面的birds文件就是’/birds’）

  • req.body：包含提交到请求体中的键值对数据。默认为undefined，使用body-parsering中间件进行解析填充。

  • req.cookie：当使用 cookie-parser中间件后，为一解析后的cookie对象，如果请求中不携带cookie，则为{}。

  • req.fresh：是否是‘fresh’的，与req.stale相反。
  • req.hostname
  • req.ip
  • req.originalUrl

  • req.params：路由参数的属性对象

  • req.path：URL的path部分，example.com/users?sort=desc，为”/users”

  • req.protocol：协议

  • req.query：查询字符串

  • req.router：当前的路由

  • req.secure：等价于 ‘https’ == req.protocol

  • req.subdomains：子域的数组

  • req.xhr：请求头 X-Request-With 设置为 XMLHttpRequest则为true。表示是通过ajax发过来的请求。

方法

  • req.accepts(types)：判断是否支持指定的格式（Accept域），req.accepts(‘application/json’);

  • req.get(field)：获取请求头指定值（不区分大小写），同req.header(field)

  • req.is(type)：判断是否匹配指定类型（Content-Type域）。

express-session模块

注意：自1.5.0版本，不再依赖cookie-parser中间件工作，模块可直接通过req/res读写cookie。

关于session

目的：实现请求身份验证

步骤：

  1. 服务器端的产生Session ID
  2. 服务器端和客户端存储Session ID
  3. 从HTTP Header中提取Session ID
  4. 根据Session ID从服务器端的Hash中获取请求者身份信息

注意：

session数据并不保存在cookie本身，cookie只存储sessionID，session数据是保存在服务器端的！

session(options)中的options

  • cookie：session ID的cookie对象，默认{ path: ‘/‘, httpOnly: true, secure:false, maxAge: null }.
    • domain：指定Domain Set-Cookie的值
    • expires：指定Expires Set-Cookie的日期对象值（建议设置maxAge即可）
    • httpOnly：指定HttpOnly Set-Cookie的布尔值，默认为true，此时客户端不能通过document.cookie读取cookie。
    • maxAge：设置过期时间（毫秒）
    • path：指定 Path Set-Cookie的值，默认为’/‘根域名。
    • secure：如果设置为true，则只有HTTPS才会返回cookie给服务器。
  • genid：产生session ID的函数（第一个参数是req），默认是通过uid-safe库来生产ID的。
  • name：在响应对象中设置的session ID cookie的名称，默认’connect.sid’，可以用于区分同一个host的不同app。
  • resave：是否强制将session保存至session store
  • saveUninitialized：是否强制一个未初始化的session保存到store？？
  • secret：用于标志sessionID cookie的秘密？（This is the secret used to sign the session ID cookie）
  • store：session store实例，比如new RedisStore(redisOpt)

req.session

存储或者读取session，直接通过req.session。

方法：

  • regenerate(callback)
  • destory(callback)
  • reload(callback)
  • save(callback)
  • touch()

属性：

  • id：是req.sessionID的别名，不能被修改。
  • cookie：让我们可以操作cookie对象，比如设置req.session.cookie.expires为false，使得cookie只保留到浏览器打开期间为止。

Express中的Redis存储

关于redis

  • Redis是一个基于BSD开源的使用ANSIC语言编写、支持网络、可基于内存亦可持久化的日志型、Key-Value数据库，并提供多种语言的API.
  • 做session的持久化，做缓存层。
  • 使用数据库来集中管理session，存放Session内容，并在各个子域名跨域共享Cookies(SessionID)，即可实现为每一个子域分配一个独立的node.js Web服务器，各个服务程序均可依据sessionid从数据库中寻找到同一Session，从而实现不同Web Server中的会话同步，从而实现一定程度上的负载均衡。
  • 另外，当你的Express服务器突然重启后，用户仍然可以使用当前Cookie中的SessionID从数据库中获取到他的会话状态，做到会话不丢失，在一定程度上提高网站的键壮性。
  • 如果你的NodeJS网站上的所有缓存数据都转移到了Redis后，就可做到完全状态无关，按需扩展网站的规模。
