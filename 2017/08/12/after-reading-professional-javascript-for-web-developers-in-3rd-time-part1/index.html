JS简介

简单的输入验证—》真正的编程语言
既简单、又复杂。
Brendan Eich / 1995
微软在IE3中实现了JScript，竞争Netscape Navigator 3的JavaScript。1996

1997年，ECMA完成了ECMA-262标准，即ECMAScript语言标准。“标准化一种通用、跨平台、供应商中立的脚本语言的语法和语义”。

Web浏览器只是ECMAScript实现可能的宿主环境之一！宿主环境提供基本的ECMAScript实现，也会提供该语言的扩展（比如DOM/BOM）。其它如Node和Adobe Flash。即是说，JavaScript和NodeJS、ActionScript属于同一级别，都实现了ECMAScript。

ECMAScript标准发展

基于Netscape 提交的建议JavaScript 1.1 ，实现了ES1，且要求支持Unicode标准；
=>
ES2，并无新增，仅为了与ISO/IEC-16262保持严格一致；
=>
ES3，较大修改，新增正则等，（IE5.5-7支持），标志着ES成为了一门真正的编程语言。
=>
ES4，由于改动非常大，所以被废弃；
=>
ES5，与ES4修订的同时有一个3.1版本，最终演变成了ES5，与2009-12-3日发布，新增原生JSON等。（IE8部分，IE9+完全支持）
=>
目前的ES6/7
（其实IE在ES标准的兼容上算是做的不错的了，第一个着手实现ES5。。>.<）

DOM标准发展

首先，DOM是基于XML（扩展至HTML）和ES的API接口，是用于JS来操作文档内容的。映射为节点。这种技术另称DHTML，即无需重载页面，即可修改外观和内容。

DOM1级，DOM level 1（和CSS级别类似，也是分模块进行），在W3C推动下，1998-10成为标准，规定如何映射XML文档结构，如何操作等；
=>
DOM2级，扩充了事件、样式、遍历等接口；
=>
DOM3级，进一步扩展，支持XML 1.0等；

++在DOM标准上，IE做得不好了，IE8仍处在DOM1级中，其事件等实现都是自由扩展；IE9+才完全支持DOM3++。

关于++浮动计算的舍入误差++：
0.1+0.2并不等于0.3！
因为，二进制表示小数有位数限制，如0.1 = 1/2*0 + 1/2^2*0 + 1/2^3*0 + 1/2^4*1 + 1/2^5*1 + 1/2^6*0 + 1/2^7*0 + 1/2^8*1……
如果采用16位的话就是：0.1(十进制) = 0.0001100110011001(16位二进制)，所以，有限位数下总不可能等于0.1，误差就在此。
所以：永远不要测试特定的浮点数值。比如if(a+b == 0.3)。

数字范围：Number.MIN_VALUE，Number.MAX_VALUE，任何超过此范围的转为Infinity/-Infinity。isFinite()函数可用于检测。

NaN：任何涉及NaN的操作都会返回NaN！isNaN()函数可用于检测。

++数字转换++：Number()函数的转换规则（与一元加操作符相同）：

  • undefined：Number(undefined); //NaN
  • 字符串：

    • Number(“”); //0
    • Number(“001”); //1
    • Number(“-1.23”); //-1.23
    • Number(“0xff”); //255
  • 对象：依次调用valueOf()、toString()方法，再运用上面规则。

++parseInt()解析与Number()不同之处++：

  • parseInt(“123hello”); //123
  • parseInt(“”); //NaN
  • parseInt(“22.5”); //22
  • parseInt(null); //NaN
  • 指定进制：

    • parseInt(“11”,8); //9

操作符

一元操作符：只能操作一个值的操作符。

  • 递增、递减：++age; age–等。也可用与非数值类型，此时会先进行Number()转换，再进行加减1的操作。
  • 加、减：+num; -num。也可用于非数值类型，一元加相当于就是Number()（有时感觉还挺好用的，如+new Date），一元减多了个变为负数的步骤。

位操作符：按内存中表示数值的位来操作。ES以IEEE-754 64位格式存储数值。
对于有符号的整数，第32位表示符号，0为正，1为负。

求一个负数的二进制码：先求绝对值二进制码 => 反码 => 反码+1。但是ES会尽力隐藏这种计算，如(-18).toString()返回’-10010’。
当对数值进行位操作时：64位数值转为32位 => 位操作 => 32位结果转为64位

  • 按位非（~）：返回数值的反码，~25返回-26。
  • 按位与（&）：对应位都为1才为1，否则为0。
  • 按位或（|）：对应位有一个1即为1。
  • 按位异或（^）：对应位相同返回0，不同返回1。
  • 左移（<<）：向左移动指定位数，2 << 5返回64。左移不影响符号位，即-2 << 5返回-64.
  • 有符号位右移（>>）：与左移相反（不移动符号位）。
  • 无符号位右移（>>>）：所有32位一起移动，64>>>5返回2，-64>>>5返回134217726。

布尔操作符：

  • 逻辑非（!）：可用于任何类型值，先Boolean()，再求反。所以”!!”与Boolean()作用一样。如!!""返回false。 始终返回布尔值。
  • 逻辑与（&&）：规则：如果第一个操作数是false或者Boolean()后是false的话则返回第一个操作数（不会执行第二操作数了），否则返回第二个操作数。不一定返回布尔值。
  • 逻辑或（||）：规则：如果第一个操作数是false或者Boolean()后是false的话则返回第二个操作数，否则就返回第一个。不一定返回布尔值。（用处：var myObject = preferedObject || {};）

乘性、加法操作符：
当操作符为非数值情况下会自动进行类型转换，后台调用Number()。

  • 乘法（）：乘积超范围则为正或负Infinity；操作数中有NaN，或`Infinity0`时，返回NaN。
  • 除法（/）：类似。
  • 求模（%）：求余数。
  • 加法（+）：如果有一个操作数是字符串，那么另外一个会转为字符串，然后拼接。
  • 减法（-）：

关系操作符：
<、>、<=、>=：
规则：如果两个都是字符串，则比较字符编码；如果有一个是数值，那么另一个也转换为数值比较。（所以某些情况下，可以仅有一个数值就能开始比较了）

相等操作符：==、!=
比较前会强制转型操作数：
如果有布尔值，则布尔转为数值，true转1，false转0；
如果有数值，则都转为数值进行比较（如果是对象之类的话就先转字符串，再转数值）；
如果只有一个操作数为对象，则调用对象的valueOf()、toString()
如果两个值都为对象，则判断是否都指向同一个对象；
特殊：null和undefined是相等的，且不能将它们转换成其它任何值；

举例：

  • null == undefined; //true
  • 0 == null; //false，因为null不进行值转换
  • 0 == []; //true，因为[]转为字符串””，再转为数值0
  • 0 == {}; //false，因为{}转为字符串”[object Object]”，再转为NaN
  • 0 == [0]; //true，因为[0]转为字符串”0”，再转为0
  • 0 == [0,0]; //false，因为[0,0]转为字符串”0,0”，再转为NaN
  • [] == “”; //true，因为只有一个对象，[]转为””
  • [] == String(“”); //false，因为两个都是对象，但不引用同一个

等等等等。。。一定从转换的规则来考虑。

全等操作符：===、!==（推荐使用！）

执行环境、作用域

执行环境：定义了变量或函数有权访问的其他数据。每个执行环境都有一个与之关联的变量对象（variable object），里面保存着环境中定义的所有变量和函数。编写的代码无法访问。

浏览器中，全局执行环境被认为是window对象。某个执行环境中的所有代码执行完后，该环境被销毁，其变量对象也被销毁。全局执行环境直到程序退出时被销毁。

每个函数都有自己的执行环境。当进入一个函数时，其执行环境就会被推入一个环境栈中，函数执行完毕后被弹出，把控制权让给之前的执行环境。

代码在一个执行环境的执行过程中，会创建变量对象的一个作用域链（即++作用域链是一个指向各环境的变量对象的指针列表++）。其用途是，保证对执行环境有权访问的所有变量和函数的有序访问。
作用域链的前端是当前执行环境的变量对象，其次是包含环境的变量对象，一直延续到全局执行环境的window对象。标识符解析就是按照这个顺序进行搜索的，最终window中还找不到就报错！
如果执行环境是函数，则将其活动对象（activation object）作为变量对象。活动对象一开始只有一个arguments对象。

延长作用域链：本质是代码执行时在其作用域前端临时增加一个变量对象。如with语句。

无块级作用域：if语句，for语句等的括号、花括号并不建立局部执行环境。

垃圾收集（GC）

若不能及时释放内存，就会造成所谓的内存泄漏。
JavaScript具有自动垃圾收集机制，执行环境负责管理内存。GC必须跟踪哪些变量有用哪些变量没用。

标记清除（现浏览器所使用的）：以某种方式标记为“进入环境”和“离开环境”。离开的就可以销毁了。

引用计数：跟踪每个值被引用的次数，为0时说明可以销毁了。但在循环引用中会出问题。

出于安全考虑，系统分配给web浏览器的可用内存数量通常要比分配给桌面应用程序的少。所以在开发过程中，最好通过设置null来释放大多数的全局变量，这叫做解除引用（dereferencing）。本质是让值脱离执行环境！

  • ignoreCase：布尔值，是否有i
  • multiline：布尔值，是否有m
  • lastIndex：整数，开始搜索的字符索引。（注意：设置全局g的正则每次搜索后会自动增加lastIndex，或者最后一次未搜索到重置0。【IE-bug】IE8-即使在非全局下也会变化lastIndex值！）
  • source：字符串，正则的字面量字符串形式（并不会双重转义）

实例方法：

exec()
结合捕获组，很强大！
传入字符串参数，返回一个数组，或者未匹配到返回null
数组内容：[与整个模式匹配的字符串，第1个捕获组，第2个捕获组，第n个捕获组，index：匹配项的索引，input：传入的字符串]
若设置了全局g，则每次调用exec方法后自动增加lastIndex值。

test()
返回布尔值。简化版的 ( reg.exec(str) !== null )

函数的属性

  • length：参数的个数
  • prototype：原型，实现继承，无法for-in出其属性
  • apply()、call()：设置函数内部的this值。区别是前者的第二个参数为一数组或arguments对象。如：sayColor.call(obj)，会直接执行sayColor()。
  • bind()：绑定函数的this到指定值，（只是）创建一个函数实例。（IE9+）。如：var objSayColor = sayColor.bind(obj)，bind时并不会执行，只是生成一个函数实例。

基本包装 类型

Boolean、Number、String
包装类型是为了便于对基本数据类型调用方法。

所有包装类型值都转为true！包括：new String("")、new Boolean(false)，所以，建议不要使用Boolean对象！

Number的方法：

  • toFixed()：以几位小数来显示（四舍五入），适合货币处理。
  • toExponentical()：以几位小数的指数表示法来显示
  • toPrecision()：自动选择合适的计数方法，参数表示几位有效数字（和上面两方法参数不一样哈）

String的属性和方法：

  • length：
  • charAt()、charCodeAt()：获取传入参数位置的字符或字符编码。也可通过方括号语法访问：”abc”[1]返回”b”（IE8+支持，否则返回undefined）
  • concat()：（不改变原字符串）拼接字符串，可接收多个参数。但在实践中更多的还是加号+
  • slice()、substring()、substr()：（不改变原字符串）截取字符串。都有两个参数。前两个方法为从第一个位置截取到第二个位置。区别在于slice()参数可以为负数。最后一个方法为从第一个位置截取指定数目的字符串，第一个参数可以为负数。
  • indexOf()、lastIndexOf()：参数为一字符串，返回位置，未找到返回-1；可以添加第二个参数，即指定从哪个位置开始搜索（ES3就存在，数组的这个方法ES5才实现）
  • trim()：ES5新增，创建一个副本，删除前后空格。
  • toLowerCase()、toUpperCase()
  • localeCompare()：比较字符串，返回正数或负数或0。（直接用大于小于比较方便些吧？）
  • fromCharCode()：String.fromCharCode(104,111) //“he”
  • match()、search()、replace()、split()：与正则相关
    match()本质与exec()相同！即：var r = str.match(pattern)与var r = pattern.exec(str)返回结果是一样的。
    search()返回查找到的第一个匹配项索引，没有就返回-1。（test()方法返回直接布尔值）
    replace()接收两个参数：第一个为字符串（此时仅替换找到的第一个子字符串）或正则；第二个参数为字符串或函数，如果为字符串，则还可以使用一些特殊序列（$’、$`、$&、$1…），如果是函数，则函数的参数依次为：匹配项，捕获组1，捕获组n，匹配项索引，原始字符串。

split()接收两个参数：第一个为一字符串或正则（可有捕获组），第二个参数（可选的）为指定返回的数组元素不能超过的个数。

单体内置对象

Global 对象

web浏览器的window对象扮演着global对象。除了parseInt()、isNaN()等方法，还有：

  • encodeURI()、encodeURIComponent()：对URI进行编码，用特殊的UTF-8编码替换无效字符。区别：前者不会对本身属于URI的特殊字符编码，如冒号、正斜杠、问号、井号等，后者会对所有非标准字符进行编码。一般我们对传递参数编码使用encodeURIComponent()。
  • decodeURI()、decodeURIComponent(): 它们用来替代已被废弃的escape()和unescape()的，因为原来的方法只能正确编码ASCII字符。
  • eval()：相当于一个JS/ES解析器。注意：其创建的变量和函数不会提升；严格模式外部访问不到eval()中创建的变量；强大而危险的方法。
