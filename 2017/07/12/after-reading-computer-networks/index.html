【阅读笔记】《计算机网络》

发表于 2017-07-12 | 分类于 前端

说在前面

Metcalfe定律：网络的价值与联网的用户数的平方成正比。

这个世界之所以能被每个人挂在嘴边侃侃而谈，是因为这是个网络的世界。网络是建立在计算机的基础上，手机也好，电脑也好，就是人与人交互的载体。可见，网络的伟大之处在于真正互联了这个世界。有幸于其中且进一步去了解它，但是，真的好难！看看这本砖头一样的《计算机网络》就知道了，还仅仅是冰山一角而已。
搞前端其实也就是跟网络打交道，没网络，何来请求，何来交互。所以，还是很愿意去多了解网络的一些知识。

很经典的第五版《计算机网络》/《COMPUTER NETWORKS》，看得懂的不是很多，尤其是关于技术层面的算法之类也是一眼带过，要真正了解技术本身怕是也不是本书可以达到的目的。大致阅读了一遍，希望能理一个轮廓出来，也算上第一遍的目的了。

引言开始了

大量相互独立但彼此连接的计算机共同组成的系统称为计算机网络，如何++设计并组织++这些网络就是本书的主题。

计算机网络的使用

  • 资源共享、通信、电子商务
  • 对等（P2P）、即时消息、射频识别（RFID）
  • 移动无线、蜂窝3G/4G、GPS、进场通信（NFC）

硬件层面

两个维度来看网络：传输技术和网络尺度讨论。

传输技术：点到点（单播）、广播/组播

网络尺度：

  • 个域网（PAN, Personal Area Network）：如蓝牙
  • 局域网（LAN, Local Area Network）：会用到路由器（无线LAN）或交换机（有线LAN）等
  • 城域网（MAN, Metropolitan Area Network）：有线电视、WiMAX
  • 广域网（WAN, Wide Area Network）：Internet服务提供商（ISP）

软件层面

为了降低网络设计的复杂性，绝大多数网络都组织成一个层次栈，每一层都建立在其下一层的基础之上。每一层有其自己的通信协议，并且为上一层提供接口。协议和接口是重要的。

层次设计有一些关键问题：

  • 可靠性（检错、纠错、路由等）
  • 网络演进（协议分层、可扩展等）
  • 资源分配（统计复用带宽、流量控制、拥塞）
  • 如何保护网络（保密性、认证、完整性）

一般下层向上层提供的服务分为++面向连接服务和无连接服务++。

服务和协议的区别：服务是某一层向上一层提供的一组原语（操作），但不涉及如何实现这些操作。协议是一组规则。

参考模型

网络设计抽象出来就是层次模型。
两个很重要的模型：OSI参考模型和TCP/IP参考模型。前者没有被使用，但模型本身具有普遍意义；后者模型本身没有多大用处，但协议已经广为流传。

OSI参考模型：
开放系统互连（OSI, Open Systems Interconnection）参考模型，基于国际标准化组织（ISO）的提案。之所以叫做参考模型，是因为模型仅指明了每一层应该做些什么事，并没有定义每一层的服务和所用的协议。OSI一共7层：

  • 物理层：关注如何在一条通信信道上传输原始比特。
  • 数据链路层：掩盖物理层的传输错误，将比特信息封装成数据帧。
  • 网络层：数据包从源端路由到接收方。基于IP地址。
  • 传输层：接受上一层的数据，必要时分割成较小的单元，然后传递给网络层。端到端的层，基于端口号。
  • 会话层：提供各种服务，包括对话控制、令牌管理、同步功能等。
  • 表示层：关注所传递信息的语法和语义。
  • 应用层：具体的应用程序和服务。如HTTP协议用于web浏览。

TCP/IP参考模型：

  • 链路层/网络接口层
  • 互联网层：官方的数据包格式和协议为因特网协议（IP, Internet Protocol）。
  • 传输层：传输控制协议（TCP, Transport Contorl Protocol）、用户数据报协议（UDP, User Datagram Protocol）
  • 应用层：虚拟终端协议（TELNET）、文件传输协议（FTP）、电子邮件协议（SMTP）、域名系统（DNS）、HTTP等。

本书使用的模型：
混合模型：

  • 应用层
  • 传输层
  • 网络层
  • 数据链路层
  • 物理层

梳理：++物理层++规定了如何在不同的介质上以电气（或其它模拟）信号传输比特。++链路层++关注的是如何在两台直接相连的计算机之间发送有限长度的消息，并具有指定级别的可靠性（以太网和802.11是链路层协议的例子）。++网络层++主要处理如何把多条链路结合到网络中，以及如何把网络与网络连接成互联网络，以便使我们可以在两个相隔遥远的计算机之间发送数据包。网络层的任务包括找到传递数据包所走的路径（IP协议是例子）。++传输层++增强了网络层的传递保证，通常具有更高的可靠性，而且提供了数据交付的抽象，比如满足不同应用要求的可靠字节流（TCP协议是例子）。++应用层++包含了使用网络的应用程序。

物理层

基础

改变诸如电压或电流等某种物理特性的方法可用来在电线上传输信息。
时间t为自变量，f(t)为电压或电流值，就可以通过数学的手段进行建模。

傅里叶分析，n次谐波。
带宽：在传输过程中振幅不会明显减弱的频率的宽度。带宽是传输介质的一种物理特性。
信道的最大比特率与其带宽和信噪比相关。

传输介质

传输介质体现在带宽、延迟、成本、安装维护等。

  • 磁介质
  • 双绞线：两根相互绝缘的铜线组成（螺旋缠绕能显著降低电线的辐射）。
  • 同轴电缆：硬的铜芯，外面包上绝缘材料。更好的屏蔽性、更大的带宽。
  • 电力线
  • 光纤：以光脉冲传播。三个关键部件：光源（LED、半导体激光）、传输介质（超薄玻纤）、探测器。电话系统的非对称数字用户线ADSL。

无线传输

当电子运动时会产生电磁波，电磁波可以在空中传播（即使在真空）。1865年Maxwell预言了这种波，1887年Hertz观察到了。（这些人太厉害了。）

  • 无线电传输：AM调幅
  • 微波传输：100MHZ以上，频率较高，不能很好穿透建筑物，当到4GHZ时，易被水吸收（微波炉原理）。所以传输应该避开这个链路。各个国家管制着频段的使用，也开放非许可性频段，可免费使用（2.4GHZ段的802.11b/g和蓝牙就基于此）。
  • 红外传输：非引导性的红外波被广泛用于短程通信，具有方向性、便宜、不能穿过固体物体。
  • 卫星通信：同步卫星、中地球轨道卫星（GPS）、低地球轨道卫星（铱计划）

数字调制与多路复用

比特与代表它们的信号之间的转换过程称为数字调制。
信道通常被多个信号共享，这种形式称为多路复用技术。具体可分为时分复用、频分复用、码分复用等。

频分复用（FDM）：将频谱分成几个频段，每个用户完全拥有其中的一个 频段来发送自己的信号。AM调幅广播即是FDM的实例。相邻信道频谱间隔要较大，以便防止干扰。
正交频分复用（OFDM）：更有效，广泛用于802.11、有线电视网络、4G等。

码分多址（CDMA）：允许每个站利用整个频段发送信号，利用编码理论可以将多个并发的传输分离开。

移动电话系统

也称蜂窝电话，是因为一个地理区域被分成许多个蜂窝。每个蜂窝使用一组频率，不会被其任何一个邻居使用。
1G，模拟语音
2G，数字语音，主导系统为全球移动通信系统（GSM）
3G，数字语音和数据，以码分多址为基础。增强数据率的GSM演进（EDGE），或叫2.1G。
4G，长期演进（LTE）。

数据链路层

设计问题

本层涉及两台相邻机器实现可靠有效的完整信息块（称为帧）通信的一些算法，而不是物理层那样只关注单个比特传输。
该层要完成如下功能：

  • 向网络层提供一个定义良好的服务接口
  • 处理传输错误
  • 调节数据流，确保慢速的接收方不会被快速的发送方淹没。

提供的服务可分为三种：无确认的无连接服务、有确认的无连接服务和有确认的有连接服务。
数据链路层从网络层获得数据包，然后将这些数据包封装成帧以便传输。帧包括帧头、有效载荷和帧尾。成帧方式包括字节计数、字节填充和比特填充等

差错检测和纠正

传输错误非常普遍，必须知道如何处理传输错误。
纠错码、检错码、校验和、各种编码blabla

协议

物理层进程和部分数据链路层进程运行在一个称为网络接口卡（NIC, Network Interface Card）的专用硬件上。
点到点协议（PPP, Point-to-Point Protocol），SONET和ADSL链路都使用PPP

介质访问控制子层

用来确定多路访问信道（广播信道）下一个使用者的协议属于数据链路层的一个子层，该层称为介质访问控制（MAC, Medium Access Control）子层。在WLAN中尤为重要。

本章的主题是++如何在竞争用户之间分配单个广播信道++。信道可以是一个地理区域内的一部分无线频谱，也就可以是连接着多个节点的单根电缆或者光纤。
FM无线电广播是一个无线信道静态分配的例子。每个电台获得其波段的一部分。

以太网

有线局域网？
两类：经典以太网、交换式以太网（使用交换机）。

交换式以太网的核心是交换机，它包含一块连接所有端口的高速背板，是一个盒子，通常有4-48个端口。交换机只是把帧输出到该帧想去的端口，当交换机端口接收到来自某个站的以太网帧，它就检查该帧的以太网地址，确定该帧前往的目的地端口。取代了以前的集线器。

以太网的发展：快速以太网（802.3u）、千兆以太网（802.3ab）、万兆以太网

无线局域网

802.11体系结构主要是：每个客户端与一个接入点（AP, Access Point）关联，该AP又与其它网络连接，客户端发送和接收数据包都要通过AP进行。

物理层：所有802.11技术都使用短程无线电传输信号，通常在2.4GHz和5GHz频段，因为是非许可的。
802.11b运行速率在11Mbps
802.11a/g在54Mbps，基于OFDM。
MAC子层协议采用带有冲突避免的CSMA。

蓝牙（Bluetooth），爱立信，SIG，蓝牙3.0，跳频扩展，2.4GHz。

无线射频识别（RFID, Radio Frequency IDentification），智能卡，没有电池的标签，读写器。物联网的愿景

数据链路层交换

中继器/集线器/网桥/交换机/路由器/网关：
理解这些设备的关键在于它们运行的层次。在典型场景中，用户生成某些数据，然后将这些数据发送给远程机器。这些数据首先被传递给传输层，传输层会加上一个头（比如TCP头），然后将结果单元往下传递给网络层，网络层也会加上一个头，形成一个网络层数据包（比如IP包），然后，该分组再往下到达数据链路层，该层加上它自己的头和校验和（CRC），并将结果帧交给物理层传送出去，比如，通过一个LAN传送出去。
中继器和集线器工作在物理层，它们并不理解帧、数据包或帧头，只负责信号放大或集中多条线路等的作用。
网桥工作在数据链路层，连接两个或多个局域网，只把帧输送到所需要的端口，使用泛洪算法和向后学习法。
交换机是现代网桥的一种别称。
路由器工作在网络层。传输网关和应用网关分别在传输层和应用层。

网络层

设计问题

网络层关注的是如何将源端数据包一路送到接收方，是处理端到端数据传输的最底层。为了实现这个目标，网络层必须知道网络拓扑结构（即所有路由器和链路的集合），并从中选择适当的路径。
在路径中的路由器需要存储转发数据包交换。
提供给传输层的服务分为面向连接的服务和无连接的服务。

路由算法

是网络层软件的一部分，应保证正确性、简单性、鲁棒性（即健壮性）、稳定性、公平性和有效性等。
分为两种，非自适应算法，即静态路由；自适应算法，即动态路由。

满足最优化原则。有很多种算法：

  • 最短路径算法
  • 泛洪算法
  • 距离矢量算法
  • 链路状态路由算法

Internet的网络层

将整个Internet粘合在一起的正是网络层协议，Internet协议（IP, Internet Protocol）。
传输层获取数据流，并且将数据流拆分成段，以便作为IP数据包发送。理论上，每个数据包最多可容纳64KB，但实际上通常不超过1500字节（这样正好可放到一个以太网帧中）。

IPv4协议主宰着今天的Internet，但32位地址不够用了，所以现在在发展IPv6了，有128位地址。

Internet上的每台主机以及路由器都有一个IP地址，IP地址实际上不指向一台主机，而是一个网络接口。所以有多个接口的路由器会有多个IP地址。IP地址由高位的可变长网络和低位的主机两部分数据组成。其书写方式为点分十进制，32位共4个字节，每个字节写成十进制，取值0-255。如128.208.0.0/24表示有24位作为网络位置，剩下的8位作为主机位置，可以将之与子网掩码255.255.255.0进行AND操作以提取出该IP地址的网络部分。

除了IP协议，还有几个辅助控制协议。
Internet控制消息协议（ICMP）、ARP、动态主机配置协议（DHCP）

地址解析协议（ARP）：数据链路层的硬件网卡并不理解IP地址，NIC根据起48位全球唯一的MAC地址来发送和接收帧，它们对32位的IP地址一无所知。问题来了，如何将IP地址映射到MAC地址呢？
现在考虑主机A给主机B发送数据包：第一步，通过DNS系统查找B的IP地址；第二步，A构建包含B的IP地址的数据包，广播到以太网上；第三步，B将自己的MAC地址作为应答返回给A；第四步，A构建一个以太网帧，目标为B的MAC地址；第五步，B收到。整个中间还要做很多优化。

传输层

传输服务

传输层架构在网络层提供的服务之上，把数据传递服务从两台计算机之间扩展到了两台计算机的进程之间。还涉及可靠性、连接和拥塞控制。
传输层的代码完全运行在用户的机器上。
传输层上即使一个非常简单的单向数据交换过程也比网络层的交换过程复杂得多，发送的每个数据包（最终）都要被确认。
当不再需要一个连接时必须将之释放，以便释放两个传输实体内部的表空间，中断方式有两种：非对称的和对称的。

传输协议的要素

传输服务通过端口（port）来寻址。
TCP建立连接是个问题，使用三次握手机制。连接释放也是一个问题。

UDP与TCP

用户数据报协议（UDP）是无连接协议，是简单的。传输控制协议（TCP）是面向连接协议，目的是提供可靠的端到端字节流。

应用层

DNS域名系统

由ICANN组织进行分配，有超过250个顶级域名，进一步划分子域。顶级域名分两种，通用的，如com，国家或地区的，如cn。blabla

万维网

web诞生于1989年的欧洲原子能研究中心CERN。第一个图形浏览器Mosaic发布于1993.2。于是组建了网景公司。1994年建立了万维网联盟W3C，致力于web标准。

web页面的获取：
点击 》 查询DNS 》 建立TCP连接 》 发送请求 》 服务器响应 》 客户端显示 》 释放连接

cookie可用于web跟踪，但防止跨站。

如今的web应用模式就是云计算的普遍模式，即让计算从PC移动到Internet的共享服务器集群中。

产生动态web的方法：

  • 公共网关接口（CGI）：后台脚本处理，可以选择Python、Ruby、Perl等
  • HTML中嵌入脚本：服务器执行这些脚本并显示最终结果，可选择超文本预处理器（PHP）、JSP、ASP.NET等。
  • 客户端动态web：JavaScript，AJAX

HTTP：运行在TCP之上的请求-响应协议。HTTP 1.0中连接建立后浏览器只发送一个请求，之后一个响应被发回来，再然后TCP连接就被释放了。
HTTP 1.1支持持续连接。可连接重用。
请求的操作称为方法，如GET，通用形式是GET filename HTTP/1.1。每个请求都会得到一个响应，每个响应消息由一个状态行及可能的附加信息组成。
HTTP缓存机制。

内容分发

一些流行站点如YouTube等对网络性能提出了要求。没有单个服务器足以强大到或者可靠到足以应付如此令人吃惊程度的需求。
内容分发网络（CDN, Content Distribution Network），内容提供商在某些位置建立一组分布式机器。
DNS是CDN的核心，负责去找到合适的服务器。

Web代理是ISP或其它机构缓存内容，用户请求就不用每次都到内容所在的服务器了，Web代理还可用于过滤网站。

不同网络区域中的CDN节点称为镜像点。

另外一种可以分发大量内容的是对等网络（P2P, Peer-to-Peer）。BT（BitTorrent）是最流行的P2P协议。
